【重要】靜態 HTML 預覽無法顯示動畫的修復原則
問題根源：
  craftToHtml.ts 輸出的是純靜態 HTML 字串，在預覽 iframe 中執行。
  凡依賴 React runtime、ogl、motion/react 等函式庫的組件，
  靜態 HTML 完全無法使用這些依賴，動畫/互動效果會消失。

修復核心方法：「將組件邏輯移植為自包含的 Vanilla JS IIFE，直接 inline 在 HTML 輸出中」

實作步驟：
  1. 在 craftToHtml.ts 的對應 case 裡，生成一個唯一 ID（如 Math.random().toString(36).slice(2,8)）
  2. 用此 ID 在 HTML 中設置 DOM 元素（canvas、div 等）
  3. 在同一 case 裡建立 const script = `(function(){ ... })();` 字串
     - 腳本開頭用 document.getElementById(uid) 取得元素
     - 把 props 中的設定值（顏色、大小等）直接以 TypeScript 的 ${value} 插值嵌入腳本
     - 腳本完全不依賴任何外部函式庫，全部純 Vanilla JS
  4. 在 HTML return 字串中嵌入 <script>${script}<\/script>
     （注意結尾要寫 <\/script> 避免 HTML 解析器提早關閉 script 標籤）

各類型的移植策略：
  WebGL (ogl)：複製頂點/片段 shader GLSL 源碼，用原生 WebGL API 編譯並執行
               範例組件：EvilEye.tsx → craftToHtml.ts case "EvilEyeContainer"
  Canvas 2D：直接將原始 class/函式邏輯翻譯成 prototype-based vanilla JS
             範例組件：PixelCard.tsx → craftToHtml.ts case "PixelCardComponent"
  motion/react 動畫：用 CSS keyframes + 可選的 IntersectionObserver/mouseevent 實現
                    範例組件：DecryptedText（JS 隨機字元循環）、Stepper（JS 步驟切換）
  CSS-only：沒有互動需求時，只需 CSS animation，不需 JS
             範例組件：MorphingText、RetroGrid、FlickeringGrid

注意事項：
  - script 字串內避免出現反引號（改用單引號或 .join()）
  - 若 JS 字串含有特殊字元（如換行、單引號），需在 TypeScript 層先做 .replace() 跳脫
  - 每個實例用唯一 ID，確保同一頁面多個同類組件不互相干擾
  - WebGL canvas 建議設 alpha:false 並直接繪製背景色，避免 iframe 透明度合成問題
